# Porra Mundial 2026

## Overview
World Cup 2026 prediction/pool app. Users register, predict group stage positions and knockout bracket winners, earn progressive points based on actual results.

## Tech Stack
- Next.js 14 App Router, React 18, TypeScript
- Prisma ORM with PostgreSQL
- NextAuth.js v4 (email/password credentials)
- Tailwind CSS, Framer Motion, Lucide icons, Sonner toasts

## Visual Style
- **Theme**: Football/World Cup
- **Colors**: Green primary (hsl 145 63% 32%), Gold secondary (hsl 38 92% 50%)
- **Fonts**: DM Sans (body), Plus Jakarta Sans (display), JetBrains Mono (numbers)
- **Dark mode**: Supported via next-themes (default: dark)
- **Hero**: Stadium background image with gradient overlay

## Database Schema
- **User**: id, email, password, firstName, lastName, role (USER/ADMIN)
- **Group**: 12 groups (A-L) with 4 teams each (48 teams total)
- **Team**: name, code, flagUrl, groupId
- **Deadline**: per phase (GROUP_STAGE, ROUND_OF_32, ROUND_OF_16, QUARTER_FINALS, SEMI_FINALS, FINAL)
- **GroupPrediction**: userId + groupId → positions (JSON array of team names in predicted order)
- **BracketPrediction**: userId + round + matchIndex → teamName
- **ActualGroupResult**: groupId → positions (JSON)
- **ActualBracketResult**: round + matchIndex → teamName

## Scoring System (Configurable via Admin)
- Stored in DB `AppConfig` table (singleton JSON), editable from Admin → "Puntos y Premios" tab
- Defaults: Group exact position: 1pt, Group qualifier: 2pts, Round of 32: 3pts, Round of 16: 5pts, Quarter Finals: 8pts, Semi Finals: 12pts, Final: 20pts, Third Place: 5pts
- Prize config: entryFee (€10), split [60%, 25%, 15%], lastGetsRefund (true)
- `lib/config.ts` — getAppConfig() reads from DB with fallback defaults
- `/api/admin/config` — Admin GET/PUT, validates split sums to 100%
- `/api/config` — Public GET for authenticated users
- scoring-info.tsx and podium.tsx fetch dynamically from /api/config

## Invite Codes
- Single-use invite codes managed by admin
- Required at signup (validated server-side in /api/signup)
- __NEXT_TEST_MODE bypass for automated tests

## User Profiles
- Alias, avatar (file upload to cloud storage), editable from /profile
- UserAvatar component used in leaderboard, participants

## Password Reset
- User requests reset → admin sees in admin panel → admin sets new password → user informed

## Pages
- `/` - Landing page (redirect to dashboard if logged in)
- `/login` - Login form
- `/signup` - Registration form
- `/dashboard` - User dashboard with stats, quick actions, top 5 leaderboard
- `/predictions` - Tabs: Group Stage (drag-to-reorder) + Bracket (select winners per round)
- `/leaderboard` - Full classification table
- `/participants` - View other users' predictions
- `/admin` - Admin panel (results, bracket results, deadlines, users) - ADMIN role only

## Key Patterns
- Server components for pages, client components for interactive parts
- Flag images from flagcdn.com (w80 size)
- Predictions locked after deadline
- Admin can update actual results to trigger score calculation
- Leaderboard computed on-the-fly from predictions vs actual results

## Auth
- Admin test user seeded (john@doe.com)
- New users default to role USER
- Middleware protects dashboard/predictions/leaderboard/participants/admin routes
